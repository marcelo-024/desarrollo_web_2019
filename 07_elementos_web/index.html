Section Elements

Esto trata de elementos de seccionado, que son los que se reflejan en el esquema (outline) de la página.

Inicialmenrte estos elementos incluían body y los encabezados (h1...h6)

En HTML5 se añaden los nuevos elementos semánticos:

  • section
  • header
  • footer
  • nav
  • article
  • aside

La etiqueta Section

El elemento de HTML section (<section>) representa una sección genérica de un documento. Sirve para determinar qué contenido corresponde a qué parte de un esquema . Piensa en el esquema como en el índice de contenido de un libro; un tema común y subsecciones relacionadas. Es, por lo tanto, una etiquéta semántica.

Su funcionalidad principal es estructurar semánticamente un documento a la hora de ser representado por parte de un agente usuario. Por ejemplo, un agente de usuario que represente el documento en voz, podría exponer al usuario el índice de contenido por niveles para navegar rápidamente por las distintas partes.

Mozilla Developer

La etiqueta Header

El elemento de HTML Header (<header>) representa un grupo de ayudas introductorias o de navegación . Puede contener algunos elementos de encabezado, pero también otros elementos como un logo, una sección que aglutine secciones de encabezados, una formulario de búsqueda o cosas parecidas.

Mozilla Developer

Etiqueta footer

El elemento HTML Footer (<footer>) representa un pie de página para el contenido de sección más cercano o el elemento raíz de sección (p.e, su ancestro mas cercano article, aside, nav, section, blockquote, body, details, fieldset, figure, td).

Un pie de página típicamente contiene información acerca de el autor de la sección, datos de derechos de autor o enlaces a documentos relacionados.

Mozilla Developer

Etiqueta nav

La MDN indica que esta etiqueta crea un menú de navegación (https://developer.mozilla.org).

El elemento HTML <nav> representa una sección de una página cuyo propósito es proporcionar enlaces de navegación, ya sea dentro del documento actual o a otros documentos. Ejemplos comunes de secciones de navegación son menús, tablas de contenido e índices.

Mozilla Developer

Etiqueta article

El Elemento article de HTML (<article>) representa una composición auto-contenida en un documento, página, una aplicación o en el sitio, que se destina a distribuir de forma independiente o reutilizable , por ejemplo, en la sindicación.

Podría ser un mensaje en un foro, un artículo de una revista o un periódico, una entrada de blog, un comentario de un usuario, un widget interactivo o gadget, o cualquier otro elemento independiente del contenido.

Mozilla Developer

Etiqueta aside

El Elemento HTML Aside (<aside>) representa una sección de una página que consiste en contenido que está tangencialmente relacionado con el contenido que le rodea, que podría ser considerado independiente de ese contenido.

Estas secciones son a menudo representadas como barras laterales o como inserciones y contienen una explicación al margen como una definición de glosario, elementos relacionados indirectamente, como publicidad, la biografía del autor, o en aplicaciones web, la información de perfil o enlaces a blogs relacionados.

Mozilla Developer

Bloques neutros

La etiqueta <div> no tiene ningún valor semántico.

Unida al uso de las clases CSS es el principal elemento de diseño de la Web. Nos permite configurar el layout o distribución del espacio.

Nuevos bloques de HTML5

  • Adress.
  • Main.
  • Figure.

Según aparece en la Web de MDN , el elemento HTML <address> aporta información de contacto. Normalmente suele ser colocado dentro del elemento <footer> de la sección.

El elemento HTML <main> representa el contenido principal del <body> de un documento o aplicación. El área principal del contenido consiste en el contenido que está directamente relacionado , o se expande sobre el tema central de un documento o la funcionalidad central de una aplicación. Este contenido debe ser único al documento, excluyendo cualquier contenido que se repita a través de un conjunto de documentos como barras laterales, enlaces de navegación, información de derechos de autor, logos del sitio y formularios de búsqueda (a menos, claro, que la función principal del documento sea un formulario de búsqueda).
Mozilla Developer

Otras etiquetas relacionadas con bloques

La etiqueta <hr> crea una línea horizontal

El elemento HTML <hr> representa un cambio de tema entre párrafos (por ejemplo, un cambio de escena en una historia, un cambio de tema en una sección). En versiones previas de HTML representaba una línea horizontal. Aún puede ser representada como una línea horizontal en los navegadores visuales, pero ahora es definida en términos semánticos y no tanto en términos representativos, por tanto para dibujar una línea horizontal se debería usar el CSS apropiado.
Mozilla Developer

El elemento HTML <br> genera un salto de línea.

El elemento HTML line break <br> produce un salto de línea en el texto (retorno de carro). Es útil para escribir un poema o una dirección, donde la división de las líneas es significante. No utilices <br> para incrementar el espacio entre líneas de texto; para ello utiliza la propiedad margin de CSS o el elemento <p>.
Mozilla Developer

Text-level semantics

Hiperenlaces

La etiqueta <a>

href es el atributo principal de la etiqueta <a>, se utiliza para direccionarnos al sitio o documento al que se refiere.

Existen cuatro tipos de url (href)

  • Absolutas
  • Relativas al origen
  • Relativas al documento
  • Internas

Principales atributos: href,target y rel.

target nos permite seleccionar el destino del enlace

rel nos indica la relación entre el hiperenlace y el documento actual.

Elementos en línea neutros.

La etiqueta <span> no tiene ningún valor semántico.

Por ejemplo, a continuación quiero que la palabra verde salga de su mismo color. Para ello debo delimitar el contenido que quiero modificar dentro de <span> acompañando esta etiqueta de una clase CSS.

Elementos claramente semánticos.

  • strong
  • em
  • sub
  • sup
  • small

<strong> nos permite marcar una palabra o conjunto de ellas como importante.

<em> se utiliza para añadir énfasis linguistico a determinada palabra o frase.

<sup> y <sub> son de utilidad para establecer superíndices y subíndices. Éstos han de ser de uso generalizado y reconocibles

Ejemplo del uso de superíndices y subíndices:

H2O

E = mc2

La etiqueta small (<s>) es utilizada para introducir información adicional de carácter legal o de derechos. Es el equivalente a la llamada letra pequeña de un contrato (ejemplo en el adress sitúado en el footer del documento).

Elementos con valor semántico nuevo.

  • <b> bold
  • <i> italics
  • <u> underline
  • <s> strike

<b>: destacado con fines utilitarios

Fragmento de texto del que se llama la atención con fines utilitarios sin concederle ninguna importancia extra y sin que implique ningún cambio de tono o de estado de ánimo. E.g. palabras clave en el resumen de un documento, nombres de producto, palabras procesables en el software interactivo orientado a texto…

<i>:

calidad diferente de texto, incluyendo cambio de tono o de estado de ánimo; fragmento de texto que implique un cambio de tono o de estado de ánimo, o cualquier otra desviación de la prosa normal que en cualquier manera indique una calidad diferente de texto., E.g. una designación taxonómica, un término técnico, una frase idiomática en otro lenguaje, un pensamiento o un nombre de barco.

<u>:

Texto escrito de forma incorrecta voluntariamente; nombres propios chinos.

<s>:
